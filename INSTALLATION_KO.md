# SideClip 설치 가이드

텍스트와 이미지 클립보드 히스토리 기능을 제공하는 SideClip 크롬 확장 프로그램의 완전한 설치 및 설정 가이드입니다.

## 🚀 빠른 설치 (개발자 모드)

### 사전 요구사항
- **Google Chrome**: 버전 114 이상 (Manifest V3 및 Side Panel API 필요)
- **운영체제**: Windows, macOS, 또는 Linux
- **권한**: 확장 프로그램 설치를 위한 관리자 권한 필요할 수 있음

### 단계별 설치 방법

1. **SideClip 다운로드**
   - 저장소 복제: `git clone [repository-url]`
   - 또는 ZIP 파일 다운로드 후 폴더에 압축 해제
   - 모든 파일이 `SideClip/` 디렉토리에 있는지 확인

2. **크롬 확장 프로그램 페이지 열기**
   - 방법 1: `chrome://extensions/`로 이동
   - 방법 2: 크롬 메뉴 → 도구 더보기 → 확장 프로그램
   - 방법 3: 크롬 툴바 우클릭 → 확장 프로그램 관리

3. **개발자 모드 활성화**
   - 우측 상단의 "개발자 모드" 스위치 토글
   - 추가 개발자 옵션이 나타남

4. **확장 프로그램 로드**
   - "압축해제된 확장 프로그램을 로드합니다" 버튼 클릭
   - `SideClip` 폴더로 이동하여 선택
   - `manifest.json`이 포함된 폴더를 선택했는지 확인
   - 확장 프로그램 목록에 나타나야 함

5. **설치 확인**
   - 크롬 툴바에서 SideClip 아이콘(📋) 확인
   - 보이지 않으면 확장 프로그램 퍼즐 아이콘(⋮) 클릭 후 SideClip 고정
   - 확장 프로그램 페이지에서 "사용" 상태로 표시되어야 함

## 🎯 초기 설정 및 테스트

### 첫 설정

1. **텍스트 기능 테스트**
   ```
   1. 아무 웹페이지 방문
   2. 텍스트 선택 및 복사 (Ctrl+C 또는 Cmd+C)
   3. SideClip 아이콘 클릭 또는 Ctrl+Shift+H 누르기
   4. 복사한 텍스트가 사이드 패널에 나타나는지 확인
   ```

2. **이미지 기능 테스트**
   ```
   1. 아무 웹페이지의 이미지 찾기
   2. 이미지 우클릭
   3. 컨텍스트 메뉴에서 "📋 SideClip에 이미지 복사" 선택
   4. SideClip 열어 이미지 썸네일 확인
   5. 썸네일 클릭하여 클립보드로 다시 복사
   ```

3. **키보드 단축키 테스트**
   ```
   - Ctrl+Shift+H (Mac에서 Cmd+Shift+H): 사이드 패널 토글
   - Escape: 확인 대화상자 닫기
   ```

### 초기 설정 문제 해결

**문제**: SideClip 아이콘이 보이지 않음
- **해결**: 확장 프로그램 퍼즐 아이콘 클릭 → SideClip 고정

**문제**: 컨텍스트 메뉴가 나타나지 않음
- **해결**: chrome://extensions/에서 확장 프로그램 다시 로드

**문제**: 사이드 패널이 열리지 않음
- **해결**: 크롬 버전 114+ 확인, 크롬 재시작 시도

## 📋 일상 사용 가이드

### 텍스트 작업
- **텍스트 복사**: 평소처럼 텍스트 선택 후 Ctrl+C 사용
- **히스토리 접근**: SideClip 아이콘 클릭 또는 Ctrl+Shift+H
- **텍스트 재사용**: 히스토리의 텍스트 항목 클릭
- **텍스트 삭제**: 항목 옆의 "×" 버튼 클릭

### 이미지 작업
- **이미지 저장**: 이미지 우클릭 → "📋 SideClip에 이미지 복사"
- **이미지 보기**: 크기 정보와 함께 히스토리에 썸네일 표시
- **이미지 복사**: 썸네일 클릭하여 PNG로 클립보드에 복사
- **이미지 삭제**: "×" 버튼 또는 "모두 삭제" 사용

### 관리
- **전체 히스토리 삭제**: "모두 삭제" 버튼 (확인 후)
- **자동 정리**: 최근 50개 항목 유지
- **저장소 제한**: 이미지당 최대 5MB

## 🔧 고급 설정

### 권한 개요
SideClip의 완전한 기능을 위해 필요한 권한:

- **storage**: 클립보드 히스토리를 로컬에 저장
- **sidePanel**: 크롬 사이드 패널에 히스토리 표시
- **activeTab**: 현재 탭의 복사 이벤트 감지
- **contextMenus**: 우클릭 메뉴에 이미지 저장 옵션 추가
- **scripting**: 복사 감지를 위한 콘텐츠 스크립트 주입
- **clipboardRead**: 이미지 작업을 위한 클립보드 접근
- **host_permissions**: 이미지 다운로드를 위한 모든 HTTP/HTTPS 사이트 접근

### 저장소 정보
- **텍스트 저장소**: 크롬의 로컬 저장소 (chrome.storage.local)
- **이미지 저장소**: 대용량 파일 지원을 위한 브라우저의 IndexedDB
- **히스토리 제한**: 최근 50개 항목 (자동 정리)
- **크기 제한**: 이미지 파일당 최대 5MB
- **위치**: 로컬 기기만 (클라우드 동기화 없음)

### 성능 최적화
- 클립보드 호환성을 위한 이미지 자동 PNG 변환
- 제한 도달 시 오래된 항목 자동 정리
- 할당량 모니터링을 통한 효율적인 IndexedDB 관리
- 향상된 성능을 위한 이미지 썸네일 지연 로딩

## 🛠️ 문제 해결

### 일반적인 문제 및 해결책

**❌ 확장 프로그램이 로드되지 않음**
- 크롬 버전 확인 (114+ 필요)
- 선택한 폴더에 `manifest.json`이 있는지 확인
- 크롬 재시작 후 확장 프로그램 다시 로드 시도

**❌ 텍스트가 캡처되지 않음**
- 개발자 도구에서 콘텐츠 스크립트 주입 확인
- 웹사이트가 콘텐츠 스크립트를 차단하는지 확인
- 페이지 새로고침 시도

**❌ 이미지가 저장되지 않음**
- 이미지가 5MB 제한을 초과하는지 확인
- 외부 이미지의 인터넷 연결 확인
- 다른 이미지 형식 시도 (JPEG, PNG, GIF)

**❌ 사이드 패널이 열리지 않음**
- 크롬을 버전 114 이상으로 업데이트
- Side Panel API 지원 확인
- 키보드 단축키 Ctrl+Shift+H 시도

**❌ 컨텍스트 메뉴가 없음**
- chrome://extensions/에서 확장 프로그램 다시 로드
- 다른 확장 프로그램과의 간섭 확인
- 다른 요소가 아닌 이미지에서 특별히 우클릭

### 디버그 모드

1. **콘솔 로깅**
   ```
   1. 개발자 도구 열기 (F12)
   2. 콘솔 탭으로 이동
   3. SideClip 로그 메시지 확인
   4. 오류 메시지 확인
   ```

2. **확장 프로그램 검사**
   ```
   1. chrome://extensions/로 이동
   2. SideClip 아래 "뷰 검사" 클릭
   3. 백그라운드 스크립트 콘솔 확인
   4. 네트워크 요청 모니터링
   ```

3. **저장소 검사**
   ```
   1. 개발자 도구 → 애플리케이션 탭
   2. 텍스트 히스토리를 위한 로컬 저장소 확인
   3. 이미지 저장을 위한 IndexedDB 확인
   4. 저장소 사용량 모니터링
   ```

## 🚀 업데이트 및 유지보수

### SideClip 업데이트
1. 저장소에서 새 버전 다운로드
2. 기존 파일을 새 파일로 교체
3. chrome://extensions/로 이동
4. SideClip의 다시 로드 버튼(🔄) 클릭
5. 업데이트 후 기능 테스트

### 데이터 백업 (수동)
- **텍스트 히스토리**: 크롬의 저장소 검사기에서 내보내기
- **이미지**: 직접 내보내기 불가 (IndexedDB에 저장됨)
- **설정**: 현재 로컬에만 저장됨

### 제거
1. chrome://extensions/로 이동
2. SideClip 확장 프로그램 찾기
3. "제거" 버튼 클릭
4. 제거 확인
5. 모든 로컬 데이터가 삭제됨

```

├── sidepanel.js          # 사이드 패널 기능 및 UI
├── imageDB.js            # 이미지용 IndexedDB 관리
├── validate.js           # 입력 유효성 검사 유틸리티
├── icons/                # 확장 프로그램 아이콘 (모든 크기)
└── docs/                 # 문서 파일
```
